# Detecting AI-Generated Italian Texts

Project for the **Computational Linguistics II** course (Digital Humanities, University of Pisa, academic year 2025/2026).

The task is **binary classification of Italian texts**: distinguishing human-written texts (class 0) from texts generated by language models (class 1). The data comes from the **DeSegMa-it** shared task (subTask A). The problem is tackled with four approaches of increasing complexity, from classic machine learning with engineered features up to fine-tuning a transformer.

## Results

Accuracy and macro F1 on the test set (1000 balanced documents):

| # | Approach | Accuracy (val) | Accuracy (test) | Macro F1 (test) |
|---|----------|:---:|:---:|:---:|
| 1 | Linear SVM + n-grams (best config: word unigrams) | 0.97 | 0.88 | 0.88 |
| 2 | Linear SVM + non-lexical linguistic features (Profiling-UD) | 0.94 | 0.88 | 0.88 |
| 3 | Linear SVM + word embeddings (itWaC) | 0.93 | 0.80 | 0.79 |
| 4 | Fine-tuned BERT (`dbmdz/bert-base-italian-cased`, 3 epochs) | 0.99 | **0.95** | **0.95** |

The drop between validation and test for approaches 1–3 suggests a distribution shift between the two dataset partitions; BERT is the model that generalizes best. The full analysis (configurations tested, cross validation, most informative features, loss curves) is in the [report](Report_LCII.pdf) (in Italian).

## Repository structure

| File | Contents |
|------|----------|
| [`0_creazione_corpus.ipynb`](0_creazione_corpus.ipynb) | Construction of the training set (2000 docs), validation and test sets (1000 docs each, balanced) from the shared task data |
| [`1_svm_ngrams.ipynb`](1_svm_ngrams.ipynb) | Linear SVMs on word, lemma and PoS n-grams (1–3) and character n-grams (3–5), model selection via 5-fold CV, coefficient analysis; extra test on ChatGPT-generated text |
| [`2_svm_feature_linguistiche.ipynb`](2_svm_feature_linguistiche.ipynb) | Linear SVMs on ~130 non-lexical linguistic features extracted with [Profiling-UD](http://linguistic-profiling.italianlp.it/) (lengths, lexical richness, PoS distributions, syntax) |
| [`3_svm_word_embeddings.ipynb`](3_svm_word_embeddings.ipynb) | Linear SVMs on document representations obtained by aggregating itWaC word embeddings (three aggregation strategies compared) |
| [`4_finetuning_bert.ipynb`](4_finetuning_bert.ipynb) | Fine-tuning of Italian BERT for 3 epochs with PyTorch, per-epoch evaluation and loss curves |
| [`Classes.py`](Classes.py) | Shared utilities: `Token`/`Document` classes, Stanza annotation, n-gram extraction and filtering, text normalization, embeddings loading |
| `TRAINUD.csv`, `VALUD.csv`, `TESTUD.csv` | Linguistic features extracted with Profiling-UD for the three partitions (aggregate numeric values only, no text) |
| [`Report_LCII.pdf`](Report_LCII.pdf) | Project report with methodology and discussion of the results (in Italian) |

## Reproducibility

The notebooks are saved **with their outputs**, so the results can be inspected without running anything.

Re-running the project requires a few resources not included in the repository:

1. **DeSegMa-it dataset (subTask A)** — not redistributable, it must be requested from the shared task organizers. The files expected by `0_creazione_corpus.ipynb` are `trainset/desegma-it.subTaskA.shared.train.*.csv` and `testset/desegma-it.subTaskA.with_labels.test.*.csv`; the notebook produces `df_train.csv`, `df_val.csv`, `df_test.csv`, which are used by the other notebooks.
2. **itWaC word embeddings** (`embeddits.sqlite`) — available among the [ItaliaNLP Lab resources](http://www.italianlp.it/resources/italian-word-embeddings/), used by notebook 3.
3. The `*_documents.pkl` files (documents annotated with Stanza) are regenerated by notebook 1.

Install the dependencies with:

```bash
pip install -r requirements.txt
```
